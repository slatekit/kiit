Concepts

Key concepts in Slate Kit

Overview

There are just a few key concepts in Slate Kit that you need to know to get up and running quickly.


Concept About More
1. Env Represents a logical environment such as local, dev, qa, prod more
2. Config Configuration settings as a thin layer on Java Properties with extra features. There is 1 config file per environment more
3. Context The AppContext is a container for common dependencies ( env, config, logger, encryptor ) and is passed to components. more
4. Cloud Slate Kit integrates with Cloud infrastructure such as File Storage and Queues. AWS ( S3, SQS ) is the default but its abstracted so you can easily customize to other providers. more
5. Apis APIs in Slate Kit are are normal methods on classes and are "protocol independent" which means they can be made accessible on the CLI ( command line ) or as an HTTP Endpoint more
6. Routes Abstracts a request whether it is on the CLI ( command line ) or a HTTP Request. This is used to build Slate Kit APIs ( which are protocol independent ). more
7. Request Abstracts a request whether it is on the CLI ( command line ) or a HTTP Request. This is used to build Slate Kit APIs ( which are protocol independent ). more
8. Result Abstracts a response and models success/failures, whether it's on the CLI (Command Line) or an HTTP Response. This is used to build Slate Kit APIs and is compatible with HTTP status codes more


1. Envs

You can setup different environments ( local, dev, qa1, qa2, etc ) for your application. This is accomplished using the Env component below. It is used in the Slate Kit App ( Base App )

Features

Typed Environment name and mode is type-safe
Validated Environment can be validated at startup
About Clear description of the environment
Defaults Choose from defaults available
Names Allow for multiple environments by name and mode

Source

package slatekit.common.envs
jar slatekit-common.jar
source slatekit.common.envs

Example

  /**
   * The available environments to choose from.
   * An environment key is built name and mode. e.g. "qa1.QA"
   *
   * Each of these environments should map to an associated env.{name}.conf
   * config file in the /resources/ directory. 
   *
   * e.g.
   * /resources/env.conf     ( common      config )
   * /resources/env.dev.conf ( development config )
   * etc ..
   *
   * @return
   */
  fun defaults(): Envs =
      Envs( listOf(
          Env("loc", Dev , desc = "Dev environment (local)"),
          Env("dev", Dev , desc = "Dev environment (shared)"),
          Env("qa1", Qa  , desc = "QA environment  (current release)"),
          Env("qa2", Qa  , desc = "QA environment  (last release)"),
          Env("stg", Uat , desc = "STG environment (demo)"),
          Env("pro", Prod, desc = "LIVE environment")
      ))

2. Config

All the different types of Apps and Sample Apps using a common configuration scheme. For each app, there is a single common configuration file env.conf and then 1 environment specific configuration file for each environment. e.g. env.dev.conf All the environment configs are located in the resources directory but you can put them and reference them anywhere. Slate Kit configs are simple abstractions over Java properties files. It is used in the Slate Kit App ( Base App ).

Features

Typed Get typed values such as bool, int, long etc.
Nullable Get nullable values such as Int?
Defaults Get default values when unavailable
Lists Get lists of basic types
Maps Get maps of basic types
Decryption Auto decrypt encrypted strings
Objects Auto load objects from config such as Db connection, Api-Keys
URIs Reference files in user directory to load objects for added security

Source

package slatekit.common.conf
jar slatekit-common.jar
source slatekit.common.conf

Example

  // Examples of just a few features...
  // 1. Typed settings
  // 2. Nullable
  // 3. Defaults 
  // 4. Lists/Maps 
  // 5. Encryption/File Refs/Inheritance
  
  // Load the env.dev.conf file from /resources
  val conf = Config("env.dev.conf")

  // Get typed value: non-nullable
  // NOTE: default value for typed returned if unavailable
  println("bool  : " + conf.getBool("bVal"))
  println("string: " + conf.getString("sVal"))
  println("short : " + conf.getShort("hVal"))
  println("int   : " + conf.getInt("iVal"))
  println("long  : " + conf.getLong("lVal"))
  println("float : " + conf.getFloat("fVal"))
  println("double: " + conf.getDouble("dVal"))
  
  // Get typed value: with default value if unavailable.
  println("bool  : " + conf.getBoolOrElse("bVal", false))
  println("string: " + conf.getStringOrElse("sVal", "abc"))
  println("short : " + conf.getShortOrElse("hVal", 0))
  println("int   : " + conf.getIntOrElse("iVal", 0))
  println("long  : " + conf.getLongOrElse("lVal", 0))
  println("float : " + conf.getFloatOrElse("fVal", 0.0f))
  println("double: " + conf.getDoubleOrElse("dVal", 0.0))
  
  // Get lists/maps
  println("list  : " + conf.getList("listValInt", Int::class))
  println("map   : " + conf.getMap("mapValInt", String::class, Int::class))
  println()
  
  // More features such as decryption, file refs and more.
  // ...

3. Context

The context is a container for global objects needed for most components. Such items include the current environment ( dev, qa, pro, etc ), config settings, a logger, database info, encryptor, the host computer, the runtime version of java, and access to the Entities that provide the ORM entity services. It is created at startup and is read-only.

Features

Container Conveniently store app level dependencies.
Immutable Immutable data can be safely passed around
Global Application level can be globally available
Portable Easily pass around a single context vs many dependencies

Source

package slatekit.core.common
jar slatekit-core.jar
source slatekit.core.common

Example

  AppContext(
      arg = Args.default(),
      env = Env("local", Dev),
      cfg = Config(),
      log = LoggerConsole(),
      ent = Entities(),
      inf = About("myapp", "App1", "Sample app", "Company1, "", "", "", "", "", "", ""),
      host = Host.local(),
      lang = Lang.kotlin(),
      enc = Encryptor("wejklhviuxywehjk", "3214maslkdf03292"),
      dirs = Folders.userDir("slatekit", "apps", "sample1")
  )
