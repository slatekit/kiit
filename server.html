Server

A fully featured Web API server build in Scala using Akka-Http
Many mobile back-end features built-in. Easily extensible.

Overview

The Scala ORM component in Slate Kit is designed with simplicity in mind. Instead of an elaborate ORM, we focus on the basic features to support single-table based mapping of objects to tables or any other underling datasource. The ORM is designed in form of an "Entities" based approach ( as you will see in the docs ). The following features are supported:


Tech

Built in 100% Scala and using Akka-Http. Our tech stack is AWS. Integration available for queues via SQS, file storage via S3 and servers via EC2 and MySql. The ORM / entities persistance layer is home-grown and built using Scala Reflection and Annotations.

See more

Approach

Follows a philosophy of "protocol independent modular APIs" where you embed regular Scala APIs to run in the server and/or in the CLI. Our server code marshalls/routes WEB API REST calls to your APIs. You simply annotate your APIs to make them available.

See docs

Setup

You can configure your Slate Server with the environment, other startup info, and APIs you want "embedded", and you are ready to have the server handle REST api calls. You can configure the port, environment, logs, and other settings in application.conf.

See docs

Routing

The server routes WEB REST calls or commands on the CLI(command line interface) to the classes and methods annotated to be publically accessible in your registered APIs. To simplify marshalling calls to your APIs, we have standardized routing conventions.

See docs

Security

Control how and when the server is booted up and also what APIs the server will "embed" or host. You register modules into the server which represent your plain old Scala APIs. You can also setup authentication and authorization of your API calls.

See docs

Features

The Slate Server has built in support to easily handle version, status, health checks, diagnostics, logging, error handling and more. You can optionally register short or long-running tasks that can be run/managed remotely. APIs are embedded into the server.

See docs

Extending

You can add as many API's as you want to the Slate Server by simply creating classes and methods with annotations and registering your API's with the Server. These APIs can be called on the command line via the CLI making them protocol independent.

See docs

CLI

Since APIs are Scala classes with annotations, it is easy to make protocol independent APIs. We provide a CLI(Command Line Interface) for making calls to your APIs via the command line by simply typing out the function/method name and inputs/parameters.

See docs

Install

Slate Server relies on the Slate.Common, Slate.Core libraries and also the Slate.Cloud and Slate.Ext libraries if you plan to queues, files, and any of the mobile backend features. Since Akka-Http is used, you will also need those libraries.

See docs

AWS

To run the Slate Server on Amazon EC2, we have detailed documentation available which show how to compile the code with the Akka-Http dependencies and also how to correctly package a zip file for distributing and installing the server on EC2.
