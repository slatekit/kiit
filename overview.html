Slate Kit

About
Slate Kit is an open-source mobile back-end and server framework for mobile and web apps.

It is pre-built architecture composed of an API Server, ORM, CLI, Background-Workers and more, a comprehensive utility library, and app features ( such as users, groups, registration ).

Goal
We are designing Slate Kit to be an open-source alternative to Parse, FireBase, and other BaaS ( Back-end as a Service ) providers for you applications. It offers the following:

  1. Host your code anywhere
  2. You own your server code and data
  3. Extendible and customizable code base
  4. Built in Kotlin for modern programming
Design
The components are built like libraries and decoupled enough that you can use as little as you want, or everything for a full ready-to-go architectural foundation. With Slate Kit, you can build your start-up idea, mobile app, pet-project, or even enterprise application.

Tech
Built for the JVM, available in Kotlin
  1. Spark: Used as the Http server for Kotlin
  2. MySql : Uses MySql for data storage ( PostGres coming later )
  3. AWS: Integrates with S3, SQS for file storage and queues.
  4. Twilio: Used for SMS / Text messages
  5. SendGrid: Used for sending email
  6. Push: Integrates with Google Cloud Messaging for Android

Slate Kit is also available in Scala but is not stable yet ( Coming later )
Deploy
This is open-source, so you can deploy and host your mobile back-end anywhere:

  1. Cloud ( AWS, Azure, Google, Digital Ocean )
  2. Intranet: Run in a private network
  3. Locally: On your personal computer
Parts
Slate Kit is reasonably modular. Use as little or as much as you need.

  1. Utilities: Over 20+ utilities that can be used for any application.
  2. Architecture: Over 12+ architectural components available ( API Server, CLI, ORM, and much more )
  3. Features: Several features available out of the box such as Users, Registration, Invites, etc..
Uses
This is essentially a private BaaS ( Backend as a Service ), tailored for:

  1. Startups: Use a strong base architecture for start-up ideas
  2. Mobile Apps: Build the back-end for your mobile apps
  3. Enterprise: Build enterprise applications on the JVM with a modern tool-kit and languages
  4. Personal: Learn Kotlin/Scala, functional programming, and best practices for self-improvement
Learn
Slate Kit is designed with simplicity and maintainability in mind. It is fairly straight-forward to learn and we have samples, guides, reference examples available.
  1. Guides: See the guides for App, API, ORM, Server More docs coming soon.
  2. Sample Apps: 4 Sample apps available: Core ( models/services ), Batch, CLI, Server
  3. Examples: Examples of all the utility and architectural components available.
Cost
The architecture components, utility libraries are free. The soon to be available app features and client-side ( java/kotlin/swift mobile client libraries ) will be mostly free and/or freemium ( we are still figuring this out ), but we do plan on making things as open-source as possible. A web-based dashboard will however be a premium feature. We will also be offering some premium services to start-ups ( coming soon ).

Components

Slate Kit is modular and contains many reusable components ( see list below ). One of the main design philosophies has been to build protocol independent API's that can be "embedded" into a Web API Server or "embedded" into a CLI (Command Line Interface) Shell.

Component Visit About Description
App Base Application A powerful base app with support for command line args, environment selection, configs per environment, logging, life-cycle events, encryption, diagnostics and more
API Protocol Independent APIs Easily build protocol independent APIs using simple classes/methods with annotations, that can be hosted / run on both Slate Command Line Shell and/or as Web APIs in the Slate Server
CLI Command Line Interface A command line interface shell that you can extend to host your "Protocol Independent APIs" and/or hook into to handle and execute any user commands in a shell.
ORM Domain-Driven ORM A simple, light-weight, Domain-Driven ORM to map your entities to and from database tables. MySql is currently supported with support for MS Sql Server coming soon.
Server Web API Server A Web API server built using Akka Http that can host your "Protocol Independent APIs". You can extend this server and register your own APIs.
Architecture Architecture Components Architectural abstractions and implementations for Files, Queues, Tasks and more. Files and Queues are abstracted with default implementations for AWS S3, SQS.
Utilities Utility Components Many useful utility components and code that can be used for any application. All these are located in the Slate.Common project and independent and modular.
Mobile Mobile / Web Features Mobile and web features for most applications, such as Users, Devices, Registration, Invites, Settings and more.

Philosophy

Simplicity above all else


Tech Built for the JVM, Linux/Windows, with Cloud Integration, and designed to be deployed and hosted anywhere.
Design Library based approach instead of a "Framework". Use as little or as much as you need.
Standards Check our coding standards for kotlin and scala
Approach Simplicity and Practicality above all else
Components Composable, single-purpose components as building blocks. Organized into various projects.
Portable Designed for low vendor lock-in via a "library" based approach. Easy to get out of this tool-kit
Minimal Dependencies on external, 3rd-party libraries are kept to a minimum
Open Source Hybrid Open-Source. Majority is open-source except for some parts of the SlateKit.Ext project containing the mobile / web features
License Apache 2.0 : Please check our git repo for more info
Style Emphasis on immutability and functional programming as much as possible.
However, this is NOT a 100% purely functional code base.
We also carefully avoid certain language features in specific times/places in favour of simple, understandable, maintainable code.
