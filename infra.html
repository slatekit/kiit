Slate Kit

Architecture

Pre-built architecture saves you time and costs, so you can focus on your business.

Core architecture components

The slate.core project contains abstractions and implementations for all components below.
Queues, file storage, data access are built for AWS, Linux, MySQL


APIs

An protocol independent approach to building APIs in Slate so they run as Web APIs or on the CLI. See concepts, example and sample apps.

App

A base App for Console, CLI, Batch, or Server apps with support for environments, logging, configs and more. See concepts, example and sample apps.

CLI

Command line access to your services, apis, and/or admin features. Extend the Shell app, register apis, and have the shell marshal commands via reflection.

ORM

Light-weight domain-driven ORM to map Scala entities to tables. And a database utility for querying, updating data. See ORM, example and sample apps.

Files

A simplified interface and abstraction for any type of persistent file storage. We offer built in support for AWS S3 to store files

Notifications

A push notification abstraction primarily for mobile based push notifications. This component is a work-in-progress and not currently available.


Queues

A simplified interface and abstraction for any type of persistent queue. We offer built in support for AWS SQS as well as a simple in-memory queue.

Server

A base Web Server to host your protocol independent APIs with built in support for many features. See concepts, example and sample apps.

Tasks

Background tasks/worker/jobs pattern to handle 1 time, long-running or continuous background processing. Built in support for queue processing available.


All items can be used independently

Projects

  • slate.core
  • slate.cloud
