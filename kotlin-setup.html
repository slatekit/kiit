1. Overview

Here is an overview of all the software required to run Slate Kit ( which is built in Kotlin ). Also listed are suggested tools for convenience


Required

All the software below is required to run Kotlin and Slate Kit. Kotlin is dependent on Java. Gradle is used to build and package Kotlin projects and is the build tool for Slate Kit.

Tech Version About Link
Java 1.8 Java see below
Kotlin 1.1.2 Kotlin see below


Suggested

The software below is recommended but not required. Slate Kit supports MySql with support for PostGres and Sql Server coming later. Having the MySql Java Connector will help with the sample apps and is also needed for the ORM ( entities ).

Tech Version About Link
IntelliJ 2017.2 IDE download
Gradle 3.5 Build tool see below


Optional

Slate Kit supports building Web APIs using Spark Java, Cloud Services ( Files, Queues ) using AWS and databases using MySql. The following are needed if you plan on using any of these.

Tech Version About Link
MySql 5.7 Database Server download
MySql Connector 5.7 For database connectivity and orm download
Spark Java 2.6 For Slate API Server visit
AWS Sdk 1.10.55 File Storage( S3 ), Queues( SQS ) download


2. Installation

Installation of Java, Kotlin, Gradle and Slate Kit is fairly easy. We recommend the following installation steps and centralized location for simplicity.

Tech Version Download Instructions
Java 1.8 download
  • install java 1.8.0 to example: C:\tools\Java\jdk1.8.0
  • add home env variable: JAVA_HOME=C:\tools\Java\jdk1.8.0
Kotlin 1.1.2 visit kotlin
  • install Kotlin via the gradel plugin
  • add "org.jetbrains.kotlin:kotlin-gradle-plugin:1.1.2" to your build script
  • refer to ROOT\src\lib\kotlin\slatekit-common\build.grade for more info
Gradle 3.5 download
  • install Gradle from download url below to c:\tools\gradle
  • add C:\tools\gradle\bin; to your system PATH variable.
  • you can also use the gradle-wrapper.jar if you are familiar with gradle


3. Setup

Slate Kit is available in 3 different ways. You can use maven references from gradle, manually download from bintray or download the releases which contain both the binaries and sample apps.


a) Bintray

You can manually download just the binaries from bintray.

See Bintray for just the binaries


b) Releases

You can download the release which contains both the binaries and sample apps

See Releases for binaries + samples


c) Gradle

You can add references in your gradle scripts like this


buildscript {
    ext.kotlin_version = '1.1.2'

    repositories {
        mavenCentral()
    }
    dependencies {
        classpath "org.jetbrains.kotlin:kotlin-gradle-plugin:$kotlin_version"
    }
}

apply plugin: 'java'
apply plugin: 'kotlin'

repositories {
    mavenCentral()
    maven {
        url  "http://dl.bintray.com/codehelixinc/slatekit"
    }
}

dependencies {
    compile "org.jetbrains.kotlin:kotlin-stdlib:$kotlin_version"
    compile "org.jetbrains.kotlin:kotlin-stdlib-jre8:$kotlin_version"
	
    // Reference the slate kit binaries here
    compile 'com.slatekit:slatekit-common:0.9.5'
    compile 'com.slatekit:slatekit-meta:0.9.5'
}

6. Compiling

Option 1: Using binaries from a release

If you want the binaries + sample apps, download the releases.

See Releases for binaries + samples and Bintray for just the binaries

Step About Eg.
1 Download the release as a zip file slatekit_kotlin_v_0_9_5.zip
2 Extract to any location c:\tools\lib\slatekit\0_9_5
3 Reference all the slatekit_* jars in the bin directory c:\tools\lib\slatekit\0_9_5\bin
4 Reference dependencies in your build.gradle or IntelliJ see above ( json-simple, aws sdk, spark-java )


Option 2: Compiling Slate Kit yourself

Building Slate Kit is simple using Gradle.

See SlateKit build.gradle and settings.gradle

Step About Eg.
1 Ensure you have gradle gradle --version
2 Download Slate Kit c:\tools\lib\slatekit\0_9_2
3 Open terminal and cd to \src\lib\kotlin c:\tools\lib\slatekit\0_9_2\src\lib\kotlin
4 Build using gradle gradle build
5 Copy Slate Kit jars to a central dist directory gradle copyJars

build

	
  c:\tools\lib\slatekit\src\lib\kotlin>gradle build

jars

	
  c:\tools\lib\slatekit\src\lib\kotlin>gradle copyJars


7. Running

You can run your kotlin apps just like how you would run java apps. The only thing to keep in mind is to ensure that the Kotlin runtime libraries are available in your classpath.

Notes

  1. Ensure kotlin-stdlib.jar
  2. Ensure kotlin-reflect.jar
  3. Ensure kotlin-stdlinb-jre8.jar
  4. Ensure mysql-connector-java.jar if using a database or the Slate Kit Entities module
  5. Ensure spark-core.jar and its dependencies if using the Slate Kit Server module
