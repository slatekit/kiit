Slate Kit

Setup

Setup Slate Kit for rapid development.

Slate Kit will soon be available on Maven


Required

All the software below is required to run Kotlin and Slate Kit. Kotlin is dependent on Java. Using Gradle will making building and packaging your apps easier.

Tech Version About Link
Java 1.8 Java see below
Kotlin 1.1.2 Kotlin see below
Gradle 3.5 Build tool see below


Suggested

The software below is recommended but not required. Slate Kit supports MySql with support for PostGres and Sql Server coming later. Having the MySql Java Connector will help with the sample apps and is also needed for the ORM ( entities ).

Tech Version About Link
IntelliJ 2017.2 IDE download
MySql Connector 5.7 Database connectivity download


Optional

Slate Kit supports building Web APIs using Spark Java, Cloud Services ( Files, Queues ) using AWS and databases using MySql. The following are needed if you plan on using any of these.

Tech Version About Link
Spark Java 2.1 ( 2.5 very soon ) For Slate API Server visit
AWS Sdk 1.10.55 File Storage( S3 ), Queues( SQS ) download
MySql 5.7 Database Server download


Installation

Installation of Java, Kotlin, Gradle and Slate Kit is fairly easy. We recommend the following installation steps and centralized location for simplicity.

Tech Version Download Instructions
Java 1.8 download
  • install java 1.8.0 to example: C:\tools\Java\jdk1.8.0
  • add home env variable: JAVA_HOME=C:\tools\Java\jdk1.8.0
Kotlin 1.1.2 visit kotlin
  • install Kotlin via the gradel plugin
  • add "org.jetbrains.kotlin:kotlin-gradle-plugin:1.1.2" to your build script
  • refer to ROOT\src\lib\kotlin\slatekit-common\build.grade for more info
Gradle 3.5 download
  • install Gradle from download url below to c:\tools\gradle
  • add C:\tools\gradle\bin; to your system PATH variable.
  • you can also use the gradle-wrapper.jar if you are familiar with gradle
SlateKit 0.9.2 download
  • download the current release from Releases
  • the binaries are located in ROOT\bin;
  • SlateKit will be available via Maven soon
  • you will need to reference the jars until Slate Kit is on Maven

Slate Server

The Slate Server is a thin layer on top of Spark Java to support the Slate Kit API approach ( protocol independent APIs that can run as Web APIs and on the CLI Command line ). SparkJava is dependent on Jettry, refer to Spark for more info.

Compiling

Slate Kit is not yet available on Maven ( work is in progress ). You can use pre-built jars available from the release or reference Slate Kit projects/modules from source code.

Notes

  1. Ensure dependent jars ( listed above ) are in your build.sbt or in your lib folder
  2. Slate Kit is NOT YET available on Maven Central, you must use the release pre-built jars or source code
  3. If you are using binaries, drop the Slate Kit jar files from the release into your sbt lib folder ( see below )
  4. If you want to use using sources, add project dependencies to the slate kit projects ( via IntelliJ )
  5. Refer to the Sample Apps for more info

Running

You can run your kotlin apps just like how you would run java apps. The only thing to keep in mind is to ensure that the Kotlin runtime libraries are available in your classpath.

Notes

  1. Ensure kotlin-stdlib.jar
  2. Ensure kotlin-reflect.jar
  3. Ensure kotlin-stdlinb-jre8.jar
  4. Ensure mysql-connector-java.jar if using a database or the Slate Kit Entities module
  5. Ensure spark-core.jar and its dependencies if using the Slate Kit Server module
